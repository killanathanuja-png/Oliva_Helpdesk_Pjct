Admin Department Flow

Step 1: Ticket Created

Helpdesk Admin raises ticket → selects Admin Department

Status: Open

Email → Raiser (Ticket Created)

▼

Step 2: Auto-Assign to L1

System: Center → City → Escalation Matrix → L1 Person

Email → L1 Assignee (Ticket Assigned)

▼

Step 3: L1 Works on Ticket

L1 reviews, adds comments, works on issue

Status: In Progress

▼

✓ L1 Fixes Issue

Goes to Resolution ▼

✗ L1 Not Resolved

SLA: 8 working hrs

Auto → L2

▼

Step 4: Auto-Escalate to L2

Status: Escalated L1 → Assigned to L2

Email → L2 User

Comment: "Auto-escalated to L2. Originally assigned to [L1]"

▼

✓ L2 Fixes Issue

Goes to Resolution ▼

✗ L2 Not Resolved

SLA: 18 hrs (8+10)

Auto → L3

▼

Step 5: Auto-Escalate to L3

Status: Escalated L2 → Assigned to L3

Email → L3 User

Comment: "Auto-escalated to L3. L2 did not resolve."

▼

✓ L3 Fixes Issue

Goes to Resolution ▼

CLOSURE PROCESS

Step 6: Helpdesk Admin → Resolved

Helpdesk Admin confirms issue is fixed

Changes status to Resolved

Email → Admin Dept Assignee (Status Update)

▼

Step 7: Admin Department → Closed

Admin Dept (L1/L2/L3) reviews & changes to Closed

Email → Helpdesk Admin / Raiser (Status Update)

SLA Working Hours: 10:00 AM – 6:00 PM IST, Mon–Sat. Sundays excluded. System checks every 5 minutes.

IT Department Flow

Step 1: Ticket Created

Helpdesk Admin raises ticket → selects IT Department

Status: Open

Email → Raiser (Ticket Created)

▼

Step 2: No Auto-Assignment

Ticket stays Unassigned

Both IT team members can see all IT tickets

Email → Both IT Members (Ticket Created)

▼

Step 3: IT Member Picks Up Ticket

An IT member self-assigns or is assigned by admin

Status: In Progress

▼

Step 4: IT Works on Issue

IT member investigates, adds comments, resolves the issue

Can request inputs: Awaiting User Inputs

▼

Step 5: Awaiting User Inputs (If Needed)

IT requests more info from Helpdesk Admin

Helpdesk Admin responds → User Inputs Received

IT resumes work

▼

✓ IT Fixes the Issue

Issue has been resolved by the IT member

CLOSURE PROCESS

Step 6: Helpdesk Admin → Resolved

Helpdesk Admin confirms issue is fixed

Changes status to Resolved

Email → IT Assignee (Status Update)

▼

Step 7: IT Department → Closed

IT Member reviews & changes to Closed

Email → Helpdesk Admin / Raiser (Status Update)

✓ TICKET COMPLETE

Key Differences from Admin:
• No auto-assignment — IT members pick up tickets themselves
• No auto-escalation — No L1 → L2 → L3 escalation
• Both IT members see all IT tickets
• Standard SLA — 24 hours (not working-hours based)
